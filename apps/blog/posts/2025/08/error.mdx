## 배경

자바스크립트는 싱글 쓰레드이기에 런타임시 에러가 발생하여 제어 흐름이 중단되면, 프로세스가 멈추기 때문에 에러처리가 매우 중요하다.

보통 이런 에처러리를 할때 에러가 날 법한 곳에 `try-catch` 구문을 감싸서 에러를 잡아내고 핸들링하여 제어흐름이 깨지지 않도록 한다.

`try-catch` 구문으로 감싸서 catch 하지 못한 에러는 어떻게 처리될까? 보통 브라우저나 Node.js 환경에서는 자동으로 Error 객체를 생성하여 throw 하고 제어 흐름을 중단시킨다.

그렇다면 Error 객체는 무엇일까?

> `Error` 객체는 런타임 오류가 발생했을 때 던져진다.
> 즉, 브라우저나, Node.js 런타임에서 에러가 발생하면 자동으로 Error 객체를 생성하여 throw 한다.
> 의도적으로 Error 객체를 확장하여 사용자 정의 에러를 만들어 사용할 수 도 있다. ( `class CustomError extends Error {....}` -> Erorr 객체를 상속하여 이를 확장 )
